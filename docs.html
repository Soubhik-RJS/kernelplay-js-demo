Getting Started

Installation

Install KernelPlay.js via npm or include it directly from CDN.

npm 
npm install kernelplay-js --save
npm install three --save
CDN

Basic Setup

Create a simple game with a scene and entity.

main.js 
import { Game, Scene, Entity } from "kernelplay-js";
import { TransformComponent, BoxRenderComponent } from "kernelplay-js";

class MyScene extends Scene {
  init() {
    const box = new Entity();
    box.addComponent("transform", new TransformComponent({
      position: { x: 400, y: 300 }
    }));
    box.addComponent("renderer", new BoxRenderComponent("red"));
    this.addEntity(box);
  }
}

class MyGame extends Game {
  init() {
    const scene = new MyScene("Main");
    this.sceneManager.addScene(scene);
    this.sceneManager.startScene("Main");
  }
}

new MyGame({ width: 800, height: 600, fps: 60 }).start();

Core Concepts

G Game

The main entry point. Manages the game loop, scenes, and rendering. Configure FPS, canvas size, and renderer type.

S Scene

Contains entities and manages updates, rendering, collisions, and raycasting. Think of it as a level or game state.

E Entity

A game object that holds components. Entities have tags and layers for organization and collision filtering.

C Component

Behaviors attached to entities. Includes transform, renderer, collider, rigidbody, and custom script components.

Components

TransformComponent

Position, rotation, and scale of an entity.

box.addComponent("transform", new TransformComponent({
  position: { x: 0, y: 0, z: 0 },
  rotation: { x: 0, y: 0, z: 0 },
  scale: { x: 1, y: 1, z: 1 }
}));

ColliderComponent

Enables collision detection and trigger events.

box.addComponent("collider", new ColliderComponent({
  width: 50,
  height: 50,
  isTrigger: false
}));

Rigidbody2DComponent

Adds physics simulation to 2D entities.

box.addComponent("rigidbody2d", new Rigidbody2DComponent({
  mass: 1,
  gravityScale: 1,
  isKinematic: false,
  drag: 0.05,
  useGravity: true
}));

Renderers

KernelPlay.js supports multiple rendering backends for different use cases.

Canvas 2D

Default renderer, best for simple 2D games.

WebGL 2D

Hardware-accelerated 2D for better performance.

Three.js 3D

Full 3D rendering with Three.js integration.

// Use WebGL 2D Renderer
const game = new MyGame({
  renderer: new WebGL2DRenderer(),
  width: 800,
  height: 600,
  fps: 60,
  backgroundColor: "#eeeeee"
});

// Use Three.js Renderer
const game = new MyGame({
  renderer: new ThreeRenderer(),
  width: 800,
  height: 600,
  fps: 60
});

Physics System

Built-in physics engine with collision detection, rigidbodies, and gravity. 

// Create a physics-enabled entity
const player = new Entity("Player");
player.addComponent("transform", new TransformComponent({
  position: { x: 400, y: 100 }
}));
player.addComponent("rigidbody2d", new Rigidbody2DComponent({
  useGravity: true,
  mass: 1
}));
player.addComponent("collider", new ColliderComponent());

// Control physics in scripts
class PlayerController extends ScriptComponent {
  update(dt) {
    const rb = this.entity.getComponent("rigidbody2d");
    
    if (Keyboard.isPressed("ArrowRight")) {
      rb.velocity.x = 200;
    }
    if (Keyboard.isPressed("Space")) {
      rb.velocity.y = -300; // Jump
    }
  }
  
  onCollision(other) {
    console.log("Collided with:", other.name);
  }
}

Input System

Keyboard Input

import { Keyboard } from "kernelplay-js";

// Check if key is held down
if (Keyboard.isPressed("ArrowRight")) { }

// Check if key was just pressed this frame
if (Keyboard.wasPressed("Space")) { }

// Check if key was just released
if (Keyboard.wasReleased("Escape")) { }

Mouse Input

import { Mouse } from "kernelplay-js";

// Mouse position
const x = Mouse.x;
const y = Mouse.y;

// Mouse buttons (0=left, 1=middle, 2=right)
if (Mouse.isPressed(0)) { }
if (Mouse.wasPressed(2)) { }

// Raycast from mouse position
const hit = this.entity.scene.raycast(Mouse.x, Mouse.y);
if (hit) {
  console.log("Clicked:", hit.entity.name);
}

Script Lifecycle

Script components have lifecycle methods similar to Unity's MonoBehaviour. 

import { ScriptComponent } from "kernelplay-js";

class MyScript extends ScriptComponent {
  // Called when entity is added to scene
  onAttach() {
    console.log("Attached to scene");
  }
  
  // Called once before first update
  onStart() {
    console.log("Starting...");
  }
  
  // Called every frame
  update(dt) {
    // dt = delta time in seconds
  }
  
  // Called after all updates
  lateUpdate(dt) {
    // Good for camera follow logic
  }
  
  // Called on collision with solid collider
  onCollision(other) {
    console.log("Hit:", other.name);
  }
  
  // Called when entering trigger collider
  onTriggerEnter(other) {
    console.log("Entered trigger:", other.name);
  }
  
  // Called when script/entity is destroyed
  onDestroy() {
    console.log("Cleanup");
  }
}

API Reference

Entity Methods

  • addComponent(name, component) - Add component to entity
  • getComponent(name) - Get component by name
  • removeComponent(name) - Remove component
  • destroy() - Remove entity from scene

Scene Methods

  • addEntity(entity) - Add entity to scene
  • removeEntity(entity) - Remove entity
  • raycast(x, y, options) - Cast ray for collision
  • findEntityByTag(tag) - Find entity by tag

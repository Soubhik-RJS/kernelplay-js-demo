Renderers

KernelPlay.js supports multiple rendering backends for different use cases.

Canvas 2D

Default renderer. Best for simple 2D games. Zero setup required.

WebGL 2D

GPU-accelerated 2D. Handles thousands of sprites smoothly.

Three.js 3D

Full 3D rendering with Three.js integration.

Canvas 2D (Default)

No renderer config needed ‚Äî it's used automatically if you don't pass a renderer option.

js 
// Default Canvas 2D ‚Äî just run the game
new MyGame({{ width: 800, height: 600, fps: 60 }}).start();

WebGL 2D Renderer

js 
import {{ WebGL2DRenderer }} from "kernelplay-js";

new MyGame({{
  renderer: new WebGL2DRenderer(),
  width: 800,
  height: 600,
  fps: 60,
  backgroundColor: "#eeeeee"
}}).start();

Three.js 3D Renderer

js 
import {{ ThreeRenderer }} from "kernelplay-js";
import * as THREE from "three";

new MyGame({{
  renderer: new ThreeRenderer(),
  width: 800,
  height: 600,
  fps: 60
}}).start();

// Use 3D components with Three renderer
entity.addComponent("transform", new TransformComponent({{
  position: {{ x: 0, y: 0, z: -5 }}
}}));
